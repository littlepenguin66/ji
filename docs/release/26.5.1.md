Initial release. Encrypt dotfiles into a single `.ji` file, sync via WebDAV, manage multi-device access with recipient keys.

## Install

```bash
cargo install ji-cli          # binary installed as `ji`
```

Or from source: `git clone https://github.com/littlepenguin66/ji && cd ji && cargo install --path .`

## What's Included

**Core** — `ji init` / `add` / `rm` / `list` / `status` / `diff` / `pack` / `unpack` / `check` / `doctor`

**Remote** — `ji remote add` / `remove` / `list` / `test` / `files` / `delete`, `ji push` / `pull` / `sync`. WebDAV first class, SSH via `--features ssh`.

**Security** — `ji recipient list` / `add` / `remove`. age encryption (default), PGP via `--features pgp`. Atomic writes everywhere. Passwords never stored.

**Developer** — `ji completion` (bash/zsh/fish), `.jiignore`

## Highlights

- **Silent by default.** Operation commands print nothing on success. Query commands print results. Warnings go to stderr.
- **Hostname-based naming.** `ji pack` names the output `<hostname>.ji`, so each machine naturally produces a different file.
- **Multi-recipient.** Add multiple SSH keys as recipients. Any one key can decrypt the same `.ji` file — no shared secrets.
- **Doctor diagnoses everything.** `ji doctor` checks config, keys, manifest integrity, and remote connectivity in one command.
- **Code as documentation.** Zero comments in the codebase outside of clap help text. 49 tests, 0 warnings.
